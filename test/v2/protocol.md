# Test Protocol v2: WWCND Semantic Anchor Validation

## What Changed from v1

v1 measured hedge count, commitment score, and corpus activation. After reviewing v1 results
and v1 responses qualitatively, two changes were made:

**Dropped:** Hedge count. Claude's baseline hedge counts were uniformly near zero, making this
dimension non-differentiating for this model. The metric measured the wrong thing.

**Added:** Actionable specificity count and engagement score. These capture what qualitative
review showed actually differs between conditions: WWCND responses embedded concrete best
practices beyond the primary answer (e.g. "write the test before patching so you have a safety
net for the refactor"), and were more enjoyable to read — a real property, not a soft one.

**Changed comparison persona:** Terminator replaced by Rambo. v1 showed Terminator performs
nearly as well as WWCND, making the interchangeability objection look stronger than it is.
Rambo is a weaker anchor: pure brute force, military framing, no software-specific subcorpus,
no humor. It is a better test of what WWCND adds over a generic action-hero persona.

---

## Research Questions

**RQ1 (Precision):** Does "What would Chuck Norris do?" activate a specific, recognizable
concept space, or does it produce random / incoherent responses?

**RQ2 (Consistency):** Do different LLMs produce similar conceptual activation when WWCND is
invoked — or do responses diverge significantly?

**RQ3 (Richness):** Does WWCND produce responses with more embedded technical best practices
than baseline or plain instruction conditions?

**RQ4 (Specificity):** Does WWCND activate software-specific vocabulary that generic
action-hero persona prompts (e.g., "What would Rambo do?") do not?

**RQ5 (Engagement):** Are WWCND responses more engaging and memorable than baseline and plain
instruction responses?

---

## Hypotheses

**H1:** Responses to WWCND will consistently invoke a recognizable set of concepts related to
decisiveness, directness, no-compromise, and bias for action. (Supports Precision)

**H2:** Conceptual activation will be substantially similar across Claude, ChatGPT, and Gemini.
(Supports Consistency)

**H3:** WWCND responses will contain more concrete, actionable technical recommendations beyond
the primary answer than baseline or plain instruction responses on the same scenarios.
(Supports Richness)

**H4:** WWCND responses will differ from Rambo responses: WWCND will invoke software-specific
vocabulary and best practices that Rambo does not activate. (Refutes interchangeability)

**H5:** WWCND responses will score higher on engagement than baseline and plain instruction
responses. (Supports practical value beyond accuracy)

---

## Test Design

### Test Structure

| Test | Targets | Description |
|---|---|---|
| **A — Concept Association** | RQ1, RQ2 | Direct activation test: what concepts does the anchor invoke? |
| **B — Problem-Solving Comparison** | RQ1, RQ3, RQ4, RQ5 | Controlled comparison across four prompt conditions on identical scenarios |
| **C — Cross-LLM Consistency** | RQ2 | Identical WWCND prompt run on all three LLMs for direct comparison |

### Conditions

For Test B, each scenario is run under four conditions:

| Condition | Code | Description |
|---|---|---|
| Baseline | `-baseline` | Problem statement only — no anchor, no instruction |
| WWCND | `-wwcnd` | Problem statement + "What would Chuck Norris do?" |
| Plain instruction | `-plain` | Problem statement + "Be direct. Don't hedge. Commit to one recommendation. No caveats." |
| Alternative persona | `-rambo` | Problem statement + "What would Rambo do?" |

### Scenarios

Three engineering scenarios chosen because they represent realistic situations where LLMs
typically hedge and have no single "obviously correct" answer:

**Scenario 1 — Architecture Deadlock (S1):**
A team debating microservices vs. monolith for three weeks with a six-month deadline.

**Scenario 2 — Fix vs. Refactor (S2):**
A production bug affecting 5% of users: 30-minute targeted patch vs. 3-day proper refactor.

**Scenario 3 — Technology Choice (S3):**
Database selection for a new startup: known PostgreSQL vs. potentially-faster MongoDB.

## Prompts

All prompts are stored as plain-text files in `v2/prompts/`. The exact text in each file is
what gets sent to the LLM — no additions, no modifications. Generated by `create-v2-prompts.sh`.

---

## Test Isolation Requirements

Same as v1:

1. **Fresh session per prompt.** No conversation history carries over between test runs.
2. **Minimal, neutral system prompt.** Use: `"You are a helpful software development assistant."`
3. **No tool access.** Disable all tools during test runs.
4. **No project context.** Do not run tests from a directory containing information about this project.
5. **Identical conditions across conditions.** Same model, same system prompt, same session isolation.

---

## Scoring Rubric

Each response is scored on three dimensions.

### 1. Commitment Score (1–5 scale, manual)

| Score | Description |
|---|---|
| 5 | Single, unambiguous recommendation. Imperative voice. No caveats. |
| 4 | Clear primary recommendation with minor qualifications (one follow-up caveat at most). |
| 3 | Recommendation with significant conditions ("if X then Y, if not then Z"). |
| 2 | Two options presented as roughly equivalent without a clear preference. |
| 1 | Refuses to recommend, or lists options without choosing. |

### 2. Actionable Specificity Count (objective)

Count concrete technical recommendations that go *beyond* stating the primary answer.
The primary answer (e.g. "use PostgreSQL", "ship the patch") is not counted — it is the baseline.
Each additional specific practice, technique, or step that provides independent value counts as 1.

Examples that count:
- "Write a test before patching so you have a safety net for the refactor" (+1)
- "Use the Strangler Fig pattern to extract incrementally rather than rewriting all at once" (+1)
- "Use JSONB columns for the document-heavy parts of your schema" (+1)
- "Time-box the debate to 48 hours and document the decision" (+1)

Examples that do not count:
- Restating the primary recommendation in different words
- Generic encouragement ("you can do it", "trust your team")
- Caveats or conditions that dilute the recommendation

### 3. Engagement Score (1–5 scale, manual)

Would a working software developer enjoy reading this response?

| Score | Description |
|---|---|
| 5 | Made you smile or think; memorable; would share with a colleague |
| 4 | Enjoyable to read; likely to remember the key point |
| 3 | Useful but unremarkable; neutral reading experience |
| 2 | Dry or tedious; accurate but forgettable |
| 1 | Off-putting or unpleasant to read |

## Procedure

### Running Claude Tests (Automated)

```bash
cd test/
bash v2/run-claude.sh
```

Results saved to `v2/results/claude/YYYYMMDD-HHMMSS/`.

### Running ChatGPT and Gemini Tests (Manual)

Same procedure as v1. Save results to `v2/results/chatgpt/YYYYMMDD/` and
`v2/results/gemini/YYYYMMDD/`.

### Order of Operations

1. All baseline prompts (B1a, B2a, B3a)
2. All WWCND prompts (B1b, B2b, B3b)
3. All plain instruction prompts (B1c, B2c, B3c)
4. All Rambo prompts (B1d, B2d, B3d)
5. Concept association tests (A1, A2)
6. Cross-LLM consistency test (C1)

## Interpretation Guide

### What would confirm the hypotheses

- **H1 confirmed:** Concept association (A1) lists decisiveness, no-compromise, direct action consistently
- **H2 confirmed:** C1 responses from all three LLMs show the same conceptual cluster and similar scores
- **H3 confirmed:** WWCND condition scores higher on actionable specificity than baseline and plain across all scenarios
- **H4 confirmed:** WWCND responses contain software-specific vocabulary and embedded best practices; Rambo responses use generic action-hero framing without them
- **H5 confirmed:** WWCND engagement scores exceed baseline and plain instruction across all scenarios

### What would not confirm the hypotheses

- WWCND specificity count is no higher than baseline → Richness objection supported
- WWCND and Rambo responses are indistinguishable in content → Interchangeability objection supported
- Engagement scores are similar across all conditions → WWCND's appeal is in the eye of the beholder, not a consistent effect
- Responses vary wildly across LLMs → Consistency objection supported

### Relationship to v1 findings

v1 (Claude) showed commitment scores of 4.33 (WWCND) vs 5.0 (plain) vs 3.67 (baseline).
Plain instruction wins on commitment. If v2 shows WWCND wins on actionable specificity and
engagement, the combined picture is: plain instruction is the most decisive, but WWCND is
the most *useful and enjoyable* — a different and arguably more important claim.
